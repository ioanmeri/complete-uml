# Section 4: Package diagram

Package Diagram helps large complex system to make them more manageable
- a **Container** or a **Group** or **Namespace** mechanism to organise related elements
  - such as classes, interfaces, other packages, other diagrams
- structural diagram
  - can contain classes, components, use cases, other model elements
- creates a hierarchical structure of the system

---

## Basics of Package Diagram

- Package
- Subsystems
  - a group of related packages and elements / or a set of functions
- Dependencies
  - import: One package imports the functionality of another package
  - access: One package requires assistance from functions of another package

---

## Benefits of Package Diagram

- Provides a high-level view of the system
- Facilitates modular design
- Encourages reuse
- Enables team collaboration
- Simplifies maintenance
