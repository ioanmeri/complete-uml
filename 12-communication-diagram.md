## Introduction

also referred as Collaboration Diagram in older UML version

Provides the same information as shown by the sequence diagram but in a different way

Pay more emphasis on **interaction messages** between the objects or parts of the system

---

## Basics of Communication Diagram

**Objects / Lifelines**

Components or entities that participate in the communication
- Rectangle with `obj:type`

**Connector**

It is the line which connects the two objects together

**Messages**
 
Information packets that are passed between the objects or actors
- `1 : Message `
- can be of different type e.g. self message, synchronous message, asynchronous message etc

**Frame**

Rectangular frame for the diagram

Name is on the upper left corner
- e.g. `sd OnlineShoppingCommunication`

---

## Benefits of Communication Diagram

- Visual representation
- Clarity
- Communication
